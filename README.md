# Projeto Brechó

## 📌 Descrição
Este projeto consiste em um sistema web de e-commerce simulado de um brechó, permitindo navegação entre produtos, carrinho, identificação do usuário e simulação de pagamento.

O objetivo principal é aplicar conceitos de desenvolvimento web e práticas de DevOps no ciclo de vida do software.

---

## 🚀 Tecnologias Utilizadas
- HTML5
- CSS3
- JavaScript
- JSP (Java Server Pages)
- Java (estrutura do projeto)
- Git e GitHub
- GitHub Actions (CI)

---

## ⚙️ DevOps Aplicado

Neste projeto foram aplicadas práticas de DevOps com foco em automação e controle de versão:

### 🔹 Controle de Versão (Git)
O Git foi utilizado para versionamento do código, permitindo:
- Registro de alterações
- Controle de histórico do projeto
- Organização do desenvolvimento

### 🔹 Repositório Remoto (GitHub)
O GitHub foi utilizado como repositório central do projeto, garantindo:
- Armazenamento remoto do código
- Sincronização entre ambientes locais e nuvem
- Controle de versões distribuído

### 🔹 Integração Contínua (CI)
Foi implementado um pipeline simples de CI utilizando GitHub Actions.

Sempre que há um push na branch `main`, o workflow é executado automaticamente, validando o funcionamento do pipeline.

Exemplo de automação:
- Checkout do código
- Execução de comando de validação
- Confirmação de execução no ambiente CI

---

## 🔄 Pipeline (Fluxo DevOps)

Desenvolvedor → Git (commit) → GitHub (push) → GitHub Actions (CI automático)
